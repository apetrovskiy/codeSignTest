Quiz

You are supposed to design a back end for a system that can allocate phone numbers to people living in a city. The phone numbers start from 100-000-0000 and end at 999-999-9999. The system is expected to handle two types of requests:

  1. A client may request a phone number and system should allocate one of the available numbers to them.
  2. Some clients may want fancy numbers, so they can ask for a specific number to be allocated to them. If the requested number is available then the system allocates it to them, otherwise the system allocates any available number. It is expected that this will be an uncommon request.

The system is limited by RAM (1 GB) and does not have to know which number is allocated to which client. It should only keep track of which phone numbers are allocated and which are free. There are no numbers allocated initially.

Which one of the listed options is the best way to implement this system?

Store all allocated "fancy" numbers in a self-balancing binary search tree and the next phone number to be distributed.

For each phone number store its availability in a bitset.
Store the bitset index of the next to be allocated number.

For each phone number store its availability as a true or false in a hash table.

For each phone number store its availability as a true or false in a boolean array.
Store the array index of the next to be allocated number.
