Technical Software Architecture Accepting Engineering Deliverables
The Accepting Engineering Deliverables badge is designed to assess one’s ability to check if both business and technical requirements are fully and correctly implemented by engineering in a software release. A realistic use case, high-fidelity design, engineering tasks, and out engineering deliverables are provided to have a realistic working example. The checks on the deliverables vary from simple to detect design misalignments, all the way up to non-functional technical implementation gaps.
About This Assessment:

An importation function in any company that builds software is validating that both business and technical requirements specified as inputs for engineering have been correctly implemented, and that main use case is achieved, without breaking or leaving gaps in related use cases. This function is usually named “acceptance” and is usually performed by an agile product owner, a product manager, a technical team lead, a delivery manager or sometimes even a hands-on software engineering manager. This is different from the exhaustive testing that quality assurance engineers will do.

This assessment evaluates one’s ability to do such validations, by taking some sample inputs that were provided to an engineering team and asking you to validate if their deliverables fulfil the input requirements.

After you start Assessment you'll no longer be able to go back to your profile until you finish the Assessment.
Earnable Proficiencies
Stars Proficiency Description
Basic
 Able to identify most functional issues, such as use case requirements not being implemented, inputs missing or compared to the use case or design, or the outputs being broken for certain types of input.
Intermediate
 Able to identify most function issues, plus able to identify most technical (code/architecture) requirement gaps such as a clearly specified technical requirement not being met, queries not being paged / more data is brought in than actually required, etc.
Advanced
 Able to identify most obvious functional issues, plus able to identify most technical (code/architecture) requirement gaps, plus able to identify most design misalignments, such as misplaced inputs or outputs or UI not behaving as normally expected.
